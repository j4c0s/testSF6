Instrukcja dla Repeaterów

Parametry techniczne, zalecane limity przeskoku (flood) oraz profile opóźnień (txdelay) dla stacji bazowych na Zachodzie.

1. Kroki kontrolne przed montażem

Przed umieszczeniem przemiennika wysoko na dachu lub maszcie, upewnij się, że stacja działa bez zarzutu:

  • Zasilanie i pobór prądu: Sprawdź obciążenie akumulatorów lub zasilacza buforowego.
  • Poziom szumów (Noise Floor): Zmierz wskaźnik RSSI tła. Ciągły poziom szumów powyżej -110 dBm oznacza lokalne zakłócenia utrudniające odbiór. W eliminacji zakłóceń może pomóc filtr szczelinowy (cavity filter), szczególnie jeżeli w pobliżu znajdują się maszty telefonii komórkowej (BTS).

2. Nazewnictwo Repeaterów

Dobra praktyka: Ze względu na brak sztywnego krajowego standardu, najważniejsze jest podanie lokalizacji pozwalającej łatwo określić położenie przemiennika. W nazwie nie musi znajdować się sztywne określenie "typ montażu", ani skrót "RPT", o ile nazwa jednoznacznie identyfikuje fizyczną lokalizację stacji bazowej.

3. Rozgłoszenia (Adverty)

Aby zapobiec wysycaniu pasma częstymi, powtarzającymi się ramkami statusu, konfigurujemy rzadkie interwały advertów (lokalnych oraz globalnych): 

# Status lokalny tylko do bezpośrednich sąsiadów co 4 godziny (240 minut)
set advert.interval 240

# Status globalny przekazywany floodem po sieci co 23 godziny (1380 minut, 0 hopów)
set flood.advert.interval 23

Powyższe konfiguracje oznaczają, że:

  • Rozgłoszenie lokalne (advert.interval): Wysyłane jest co 240 minut (czyli równe 4 godziny). Służy wyłącznie bezpośrednim sąsiadom.
  • Rozgłoszenie globalne (flood.advert.interval): Następuje co 23 godziny (ze statusem 0 hopów) i rozprzestrzenia się po całej sieci jako flood.

4. Synchronizacja Czasu (Clock Sync)

Prawidłowy czas systemowy na repeaterze jest kluczowy dla właściwej interpretacji logów i poprawności routingu. Do synchronizacji czasu służy dedykowane polecenie: 

# Synchronizacja czasu z podłączonym urządzeniem / siecią
clock sync

Wskazówka: W przypadku, gdy czas systemowy na urządzeniu jest błędny i z jakiegoś powodu nie da się go cofnąć tradycyjną komendą, należy najpierw zresetować zegar systemowy poleceniem, a następnie wykonać synchronizację: 

# Reset zegara systemowego przy problemach z synchronizacją wstecz
clkreboot

# Następnie ponowna synchronizacja czasu
clock sync

5. Ustawienia Właściciela i Tożsamości

Dla ułatwienia diagnostyki sieciowej i identyfikacji stacji bazowych, niezwykle przydatne jest poprawne ustawienie właściciela oraz opisu stacji: 

# Ustawienie identyfikatora / znaku wywoławczego właściciela przemiennika
set owner "TWÓJ_ZNAK_LUB_NICK"

6. Pozostała konfiguracja optymalizacyjna

Poniżej opisano dodatkowe parametry optymalizacyjne, które należy wprowadzić w CLI:

  • set path.hash.mode 2: Wymusza dwubajtowy tryb generowania skrótów ścieżek (path-hash), co znacząco zmniejsza ryzyko kolizji identyfikatorów tras w większych sieciach.
  • set multi.acks 1: Aktywuje wielokrotne potwierdzenia odbioru pakietów, co poprawia niezawodność dostarczania danych przy niestabilnym linku.
  • set agc.reset.interval 500: Określa interwał (w milisekundach) resetowania automatycznej kontroli wzmocnienia (AGC) w układzie SX1262, zapobiegając zatykaniu odbiornika przez silne, bliskie sygnały.
  • set guest.password "": Pozostawia puste hasło gościa dla konsoli, co ułatwia sąsiedzką diagnostykę i weryfikację stanu stacji bez zdradzania haseł administracyjnych.
  • set flood.max 64: Maksymalny dopuszczalny limit hopów (przeskoków) dla pakietów z przypisanym i dozwolonym zakresem regionalnym (np. zachod, pl-ds).
  • set flood.max.unscoped 8: Ograniczenie przesyłu dla ogólnego ruchu bez przypisanego zakresu (unscoped). Pakiety te mają mniejszy priorytet rozprzestrzeniania się.
💡 Wskazówka dotycząca unscoped: Wartość limitu flood.max.unscoped można w pełni dostosować do swoich potrzeb poprzez uważną obserwację przychodzących wiadomości. Im mniejsza wartość, tym mniejsze obciążenie generuje ruch dla całej lokalnej sieci. Najlepiej, aby wartość ta nie przekraczała 8 (zalecany przedział to zazwyczaj 6 do 8).

Komplet poleceń optymalizacyjnych do wklejenia w CLI:

# Włączenie trybu routingu i zalecanych ustawień
set path.hash.mode 2
set multi.acks 1
set agc.reset.interval 500
set guest.password ""
set flood.max 64
set flood.max.unscoped 8

# Zapis i restart urządzenia
write
reboot

7. Profile Opóźnień Nadawania (txdelay)

Zróżnicowanie czasów opóźnienia nadawania (parametr txdelay) chroni sieć przed kolizjami radiowymi na tym samym obszarze (tzw. zjawisko ukrytego węzła).

Zasada jest prosta: im wyżej położony i lepiej słyszalny jest dany przemiennik, tym dłużej powinien odczekać przed retransmisją, aby pozwolić na nadanie stacjom położonym niżej. Liczba bezpośrednich sąsiadów również definiuje profil, do którego powinno się zakwalifikować dane urządzenie.

Profil wysokości Zastosowanie / Liczba sąsiadów Zalecane Opóźnienia CLI
WYSOKI / MASZT Wysokie wieże, maszty kratownicowe, dachy wieżowców o bardzo dużym zasięgu.
Sąsiedzi: powyżej 15 bezpośrednich sąsiadów.		 set txdelay 1.5
set direct.txdelay 1.0
set rxdelay 3
DACH / ROOFTOP Klasyczny montaż anteny na dachu domu jednorodzinnego lub niskiego bloku.
Sąsiedzi: od 5 do 15 bezpośrednich sąsiadów.		 set txdelay 0.8
set direct.txdelay 0.4
set rxdelay 3
DOM / NISKI Węzeł domowy, antena na parapecie, balkonie lub wewnątrz pomieszczeń.
Sąsiedzi: poniżej 5 bezpośrednich sąsiadów.		 set txdelay 0.3
set direct.txdelay 0.1
set rxdelay 3
